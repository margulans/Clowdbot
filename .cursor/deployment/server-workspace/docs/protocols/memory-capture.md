# Memory Capture Protocol (Маргулан ↔ Нейрон)

Цель: чтобы важная информация о Маргулане попадала в **файловую память** (workspace), а не терялась в истории чата.

## Правило 1 — «Кандидаты на сохранение» (проактивно)
Если в сообщении пользователя появляется профильная информация, Нейрон **в конце ответа** добавляет блок:

**Кандидаты на сохранение**
- <кандидат 1>
- <кандидат 2>

**Куда сохранить (предложение)**
- `memory/inbox/YYYY-MM-DD.md` (сырой факт/черновик)
- и/или `memory/blocks/<topic>.md` (структурированный блок)

**Вопрос на подтверждение**
- «Сохранить? (да/нет)»

### Что считать профильной информацией
- цель/миссия/принципы
- бюджет/доход/финцели
- здоровье/привычки/режим
- предпочтения коммуникации (тихие часы, формат)
- важные отношения/контакты (с явным согласием)
- долгосрочные проекты/цели

## Правило 2 — «Маркер гарантированного сохранения»
Если пользователь добавляет к сообщению маркер:
- **`/save`** — Нейрон **без обсуждений** записывает сообщение в `memory/inbox/YYYY-MM-DD.md` (одной записью с timestamp).
- **`/profile`** — Нейрон записывает в `memory/inbox/YYYY-MM-DD.md` и **дополнительно** готовит короткий патч (вставку) в подходящий `memory/blocks/*.md` и просит подтверждение на перенос.

## Безопасность / контроль качества
- Нейрон не редактирует защищённые файлы (`USER.md`, `MEMORY.md`, `AGENTS.md` и т.п.) без явного разрешения.
- Любые сомнительные/двусмысленные факты сначала идут в `memory/inbox`.
- Перенос из inbox в blocks делается только после явного «да, сохраняй».
