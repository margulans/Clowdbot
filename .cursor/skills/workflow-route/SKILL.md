# Маршрутизация Cursor ↔ Бот

Определи канал для задачи и выведи решение.

## Триггеры

- `старт` — при старте сессии можно напомнить о маршрутизации
- `маршрут` — явный запрос «куда это делать»
- `где это делать` — альтернативная формулировка

## Алгоритм

1. Прочитай `.ai/ROUTING.md`
2. Сопоставь запрос пользователя с таблицей типов задач
3. Выведи решение:

   **Если Cursor:**

   > Выполняй здесь. После commit и push — «Запусти /git в Telegram».

   **Если Бот:**

   > Сгенерируй ТЗ по шаблону из ROUTING.md. В конце: «Скопируй и отправь в @neironassistant_bot».

## Таблица (краткая справка)

| Тип                          | Канал  |
| ---------------------------- | ------ |
| Скиллы, .ai/, .cursor/       | Cursor |
| openclaw.json, cron, хотфикс | Бот    |

Детали — в `.ai/ROUTING.md`.
