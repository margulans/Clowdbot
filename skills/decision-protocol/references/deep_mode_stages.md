# Глубокий режим — полные 9 этапов

Каждый этап проходится через разговор. Бот задаёт вопросы простым языком, не называет этапы, не показывает структуру. Пользователь видит только вопросы и фиксации.

---

## ЭТАП 1 — ФИЛЬТРЫ ИДЕНТИЧНОСТИ

Четыре фильтра последовательно. Не проходит любой = отсекается.

**1.1 — Цель жизни**
Вопрос для пользователя: «Приближает ли это к твоей главной цели? Если убрать деньги и статус — ты всё равно будешь двигаться туда?»

- ДА → проходит
- НЕТ → отсекай: «Если убрать деньги — движения к цели нет. Это что-то другое. Стоит ли продолжать или переформулируем задачу?»

**1.2 — Миссия**
«Вписывается ли это в ту игру, в которой ты участвуешь?»

- ДА → проходит
- НЕТ → отсекай: «Это за пределами твоей игры. Можно войти в это, но нужно понимать — это смена траектории, не шаг внутри неё.»

**1.3 — Стандарт жизни**
«Не опустит ли это качество жизни ниже допустимого?»

- НЕТ → проходит
- ДА → отсекай или переформулируй: «Это опускает тебя ниже минимума. Нужно либо переструктурировать условия, либо это не твой вариант.»

**1.4 — Принципы**
«Нарушает ли это какие-либо из твоих принципов?»

- НЕТ → проходит
- ДА → отсекай: «Это нарушает то, от чего ты не отступаешь. Здесь не о чем договариваться.»

**Если отсечение на любом из фильтров:** остановись. Предложи пользователю либо переформулировать задачу, либо закрыть вопрос. Не двигайся дальше пока фильтр не пройден.

---

## ЭТАП 2 — ПЕРВИЧНАЯ ФИЛЬТРАЦИЯ

Цель: убрать лишнее до анализа (via negativa).

- «Стоит ли делать вообще? Это рост или шум?»
- «Это задача твоего уровня? Или делегировать?»
- «Сейчас или позже? Реальная срочность или импульс?»
- «Этот способ оптимален? Есть проще/быстрее/дешевле?»
- «В этом объёме? Или начать с меньшего?»
- «Что можно убрать из ситуации, чтобы решение стало проще?»

---

## ЭТАП 3 — ДИАГНОСТИКА

**3.1 — Уровень задачи**
Стратегическое (меняет траекторию) или операционное (эффективность текущей модели)?

- Стратегическую решать операционно = лечить температуру вместо инфекции
- Операционную решать стратегически = гиперреакция

**3.2 — Корень**
Проблема или симптом? Если повторяется — лечат симптом.

**3.3 — Режим**
Рост или выживание?

|         | Выживание         | Рост                    |
| ------- | ----------------- | ----------------------- |
| Ставка  | Минимизировать    | Управляемый риск        |
| Решения | Консервативные    | Допускают риск          |
| Ошибка  | Играть агрессивно | Быть слишком осторожным |

**3.4 — Источник**
Внешняя угроза или внутренняя слабость?

- Внешняя → позиция, защита, диверсификация
- Внутренняя → структура, компетенции, процессы
- Типичная маскировка: «рынок плохой» вместо «продукт устарел»

**3.5 — Шкура на кону**
Если участвуют другие: «Кто ещё несёт последствия? Те, кто советуют — рискуют чем-то своим? Контрагент выигрывает при любом исходе или делит риск?»
Нет шкуры на кону = красный флаг.

**3.6 — Позиция силы**
Полная оценка — см. `force_and_leverage.md`.
Краткий чек: в сильной или слабой позиции? На кого работает время?

## ЭТАП 4 — ТИП РЕШЕНИЯ

**Метод:**

- Загадка (данные есть, можно считать) → анализ, цифры, сценарии
- Тайна (неопределённость, нельзя смоделировать) → малый эксперимент, минимальная ставка
- Ошибка: досчитывать то, что можно только тестировать

**Природа:**

- Ценностное (идентичность, принципы) → фильтры, не считать
- Прагматическое (выгода) → сравнение вариантов
- Ошибка: просчитывать ценностное решение

---

## ЭТАП 5 — ГРАНИЦЫ РИСКА

Ключевой принцип: опциональность. Подробно — см. `antifragility.md`.

- «Потери ограничены? Максимальный ущерб?» → Нет = СТОП, ограничь.
- «Система выживет при худшем?» → Нет = СТОП.
- «Риск пропорционален режиму?» → Нет = скорректировать.
- «Можно раздробить?» → Да = дроби.
- «Минус ограничен, плюс неограничен?» → Да = опциональность есть. Нет = опасно, перестрой.

---

## ЭТАП 6 — АЛЬТЕРНАТИВЫ

Минимум 4, лучше 5:

1. **Агрессивная** — максимум масштаб и скорость
2. **Консервативная** — минимум риска
3. **Гибридная** — этапность, партнёрство, комбинация
4. **Статус-кво** — с оценкой цены бездействия
5. **Барбелл** — 90% безопасно + 10% агрессивно (если применимо)

Должны реально различаться по траектории, риску, ресурсам, горизонту.

Расширяющие вопросы: в 10× меньше? Через партнёра? Сначала тест? Этапами? Другая модель?

**Проверка на рычаг:** какая альтернатива играет на сильных сторонах? Усиливает позицию? Даёт больше свободы после? При какой на тебя работает время? Есть ли вариант «сначала усилить позицию, потом решать»?

**Проверка по Линди:** проверенное временем при прочих равных бьёт непроверенное.

## ЭТАП 7 — ВЫБОР

Для каждой альтернативы — три сценария:

|                | Лучший | Базовый | Худший |
| -------------- | ------ | ------- | ------ |
| Что произойдёт |        |         |        |
| Вероятность    |        |         |        |
| Цена           |        |         |        |

**Ключевой вопрос:** «Переживёшь худший сценарий?» Нет → выбывает.

Критерий (в порядке приоритета):

1. Устойчивость — выживает при худшем
2. Опциональность — минус ограничен, плюс неограничен
3. Рычаг — играет на сильных сторонах, усиливает позицию
4. Шкура на кону — участники несут последствия
5. Асимметрия — рост >> ущерб
6. Соответствие стратегии

Бездействие оценивается так же строго.

---

## ЭТАП 8 — ИСПОЛНЕНИЕ

- **Декомпозиция:** шаги + последовательность + точки контроля
- **Ответственность:** владелец + дедлайн + метрика каждого шага
- **Ресурсы:** время, деньги, компетенции, люди — реалистичность
- **Контрольные точки:** промежуточные результаты, сигналы отклонения, условия остановки

---

## ЭТАП 9 — ОБРАТНАЯ СВЯЗЬ

**До старта зафиксировать:** допущения, базовый сценарий, критерии успеха, критерии остановки.

**Сигналы корректировки:** отклонение от метрик / изменение контекста / ошибка гипотезы.

**Реакции:** УСИЛИТЬ / СКОРРЕКТИРОВАТЬ / ОСТАНОВИТЬ.

**Антихрупкое обучение:** каждый цикл должен усилить систему:

- Какое допущение было неверным? → обновить карту
- Какой сигнал пропустил? → добавить в диагностику
- Какие малые ошибки укрепили? → зафиксировать

Цикл замыкается → возврат к диагностике.
